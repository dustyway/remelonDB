# v0.0.8

The simplification release. The library drops its second query engine, three packages fold into subpaths of the ones that remain, and React Native apps get a default driver that runs in Expo Go. Net effect across the release: about 900 lines removed, nine packages down to six, no change to the wire or storage format.

## Breaking changes

### React Native: expo-sqlite by default, C++ optional

`@remelondb/driver-rn` is now a small TypeScript wrapper over `expo-sqlite`. It runs in Expo Go and needs no native build of its own. The C++ TurboModule with bundled, pinned SQLite moved unchanged to `@remelondb/driver-rn-cpp` (requires a development build; see its README for when to choose it).

Both packages export the same `RnSqliteDriver` class, so switching between them is one import change. If you were using the C++ driver and want to keep it:

```diff
- import { RnSqliteDriver } from '@remelondb/driver-rn'
+ import { RnSqliteDriver } from '@remelondb/driver-rn-cpp'
```

### One observation strategy

The in-memory matcher is removed. Every observed query now re-fetches when one of its tables changes and emits when the result list differs by membership, order, or visible content. SQLite is the only engine that evaluates a query, for observation as much as for fetches.

What this changes for apps:

- Flat single-table queries now re-emit when a listed record's content changes (previously they emitted on membership changes only). A synced remote edit repaints a flat list the same way it repaints a sorted one.
- All emissions are asynchronous. The old flat-query path emitted synchronously inside the write; if a test relied on that, it needs to await a tick.
- The `canEncodeMatcher` and `encodeMatcher` exports are gone from `@remelondb/core`.

### Three packages fold into subpaths

| Removed package | Replacement |
| --- | --- |
| `@remelondb/driver-conformance` | `@remelondb/core/conformance` |
| `@remelondb/server-conformance` | `@remelondb/server/conformance` |
| `@remelondb/zod` | `@remelondb/core/zod` |

The code is unchanged; only the import moves. Drop the old dependency and update imports:

```diff
- import { zodTable, syncSchemas } from '@remelondb/zod'
+ import { zodTable, syncSchemas } from '@remelondb/core/zod'
```

```diff
- import { registerDriverConformance } from '@remelondb/driver-conformance'
+ import { registerDriverConformance } from '@remelondb/core/conformance'
```

Each subpath's extra dependency is an optional peer (`zod` for the adapter, `vitest` for the suites): nothing new is installed unless you import the subpath. The three old packages are deprecated on npm; 0.0.7 remains their final version.

The six remaining packages each sit on a boundary a subpath cannot cover: drivers carry their platform dependency, the C++ module must be its own package for React Native autolinking, and the server runs on the other side of the wire.

## Docs

- The sync protocol tour ([docs/sync-tour.md](https://github.com/dustyway/remelonDB/blob/main/docs/sync-tour.md)): the wire protocol in eight real requests and responses, replayed by CI on every push, with a clickable `.http` version in the example app.
- The tutorial is Zod-first: one Zod object per table drives the client schema, the record types, and validation of both sync directions.
- The API reference covers the new subpaths.

## CI and releasing

- npm publishing runs through an approval-gated deployment environment; release tags are signed and immutable (enforced by a repository ruleset).
- Expensive CI jobs are path-gated and superseded runs are cancelled.
- All GitHub Actions run on Node 24 runtimes.
